## 本轮暴露的问题

### 1. Spec 验收清单缺失 → 实现遗漏

写了完整 spec 但没写"实现逐项验收清单"→ 设置页的 data-dir 选择器直接漏了，用户运行才发现。

**工作流改进：** Spec 阶段必须输出「实现逐项验收清单」，Implement 阶段完成时逐条对照打勾，不打勾不 commit。

### 2. 托盘菜单空白（Tauri v2 双重设置）

`setup()` 里 `TrayIconBuilder` 先传了一个菜单，`rebuild_tray_menu` 又去 `set_menu()` 覆盖 → 空白。

**根因：** 旧代码遗留 + 没验证即 commit。Implement 阶段增加了「F5 热重载验证」，如果早点验证托盘交互就能发现。

### 3. 编译错误暴露慢

`cargo check` 只在最终阶段跑，中间改完 lib.rs 后没立刻 check。Tauri 插件权限不匹配（`core:webview:allow-eval` 不存在）直到全量 build 才发现。

**改进：** 改 Rust 代码后立刻 `cargo check`（Implement 做法已有要求，执行层面需落实）。
